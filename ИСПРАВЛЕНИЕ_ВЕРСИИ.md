# Исправление версии Next.js

## Проблема:
Vercel все еще видит Next.js 15.5.7, хотя в package.json указано 16.1.1

## Решение:

### Шаг 1: Удалите старые зависимости и переустановите

```bash
cd /Users/evgeniibogatenkov/Desktop/tranio-academy

# Удалите node_modules и package-lock.json
rm -rf node_modules package-lock.json

# Переустановите зависимости
npm install
```

### Шаг 2: Проверьте версию

```bash
# Проверьте, что установлена правильная версия
npm list next

# Должно показать: next@16.1.1
```

### Шаг 3: Обновите package-lock.json в Git

```bash
# Добавьте обновленный package-lock.json
git add package-lock.json package.json

# Создайте коммит
git commit -m "Update Next.js to 16.1.1 and regenerate package-lock.json"

# Отправьте на GitHub
git push origin main
```

### Шаг 4: Проверьте на Vercel

После `git push` Vercel автоматически:
1. Увидит изменения
2. Запустит `npm install` (который установит 16.1.1)
3. Предупреждение исчезнет

---

## Если проблема сохраняется:

### Вариант 1: Принудительное обновление

```bash
# Установите конкретную версию
npm install next@16.1.1 --save-exact

# Обновите eslint-config-next
npm install eslint-config-next@16.1.1 --save-dev --save-exact

# Проверьте
npm list next
```

### Вариант 2: Очистка кэша Vercel

1. Зайдите на Vercel → ваш проект
2. Settings → General
3. Нажмите "Clear Build Cache"
4. Запустите новый деплой

---

## Проверка после исправления:

После деплоя на Vercel проверьте логи:
- Не должно быть предупреждения о 15.5.7
- Должно быть: "Detected Next.js version: 16.1.1"
