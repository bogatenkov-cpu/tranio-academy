# Очистка зависимостей и переустановка

## Проблема:
Ошибка `rm: node_modules/is-typed-array: Directory not empty` при удалении node_modules

## Решение:

### Вариант 1: Использовать более надежную команду (рекомендуется)

```bash
cd /Users/evgeniibogatenkov/Desktop/tranio-academy

# Удалить с принудительной рекурсией
rm -rf node_modules package-lock.json

# Если все еще не работает, попробуйте:
sudo rm -rf node_modules package-lock.json
```

### Вариант 2: Использовать npm для очистки

```bash
cd /Users/evgeniibogatenkov/Desktop/tranio-academy

# Очистить кэш npm
npm cache clean --force

# Удалить node_modules (если предыдущая команда не помогла)
find node_modules -type d -exec rm -rf {} + 2>/dev/null || true
rm -rf node_modules package-lock.json
```

### Вариант 3: Пошаговое удаление (если ничего не помогает)

```bash
cd /Users/evgeniibogatenkov/Desktop/tranio-academy

# Удалить только package-lock.json
rm -f package-lock.json

# Переустановить зависимости (это обновит package-lock.json)
npm install

# Проверить версию
npm list next
```

---

## После успешной очистки:

```bash
# Переустановить зависимости
npm install

# Проверить версию Next.js
npm list next
# Должно показать: next@16.1.1

# Отправить изменения в GitHub
git add package.json package-lock.json
git commit -m "Update Next.js to 16.1.1"
git push origin main
```

---

## Если проблема сохраняется:

1. **Закройте все процессы**, которые могут использовать node_modules:
   - Остановите `npm run dev` (если запущен)
   - Закройте редактор кода (VS Code, Cursor и т.д.)
   - Закройте терминалы, которые могут использовать проект

2. **Попробуйте перезагрузить компьютер** (крайний случай)

3. **Или просто обновите package-lock.json без удаления node_modules:**
   ```bash
   npm install next@16.1.1 --save-exact
   npm install eslint-config-next@16.1.1 --save-dev --save-exact
   ```
